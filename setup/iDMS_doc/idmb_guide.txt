------------
INTRODUCTION
------------

* The IDMB is a application built on top of IDMS interface to provide the
  UNIX level database accessing. It can be used to operate data as well
  as query status of data and audittrail history.


--------
START UP
--------

* IDMB is a stand alone appliction which can be run from UNIX.

* The environment variable DMSPATH need to be set to a valid DMS.PATH
  file which will be read and parsed by IDMB.

* When IDMB is run, command manual of IDMB will be displayed. Users will need
  to input the command and answer prompts to have IDMB run properly.  

* The behavior of IDMB will then base on user's inputs.

* The result or message will be shown on screen after each command.


--------------
USER INTERFACE
--------------

* User's inputs define IDMB's behaviors.

* Each prompt of IDMB will have a default value which shown in parenthesis.
  Users can choose to input a new vlaue or just hit return to use default.

  Ex: Tool Name: (idmb)

* Semicolon can be used to abort commands in any prompts.

* All inputs take wild card and list expression except the tool name.

  Example: Cell Name: (*) p12 r* ???
      	   Cell names are "p12", all cell names with "r" prefix and all 3
	   characters cell names.


--------
MESSAGES
--------

* Messages can be grouped into three categories.

	*IDMS MESSAGE* indicates a execution is successful.

	*IDMS WARNING* indicates a execution has recoverable errors.

	*IDMS ERROR*   indicates a execution has unrecoverable errors.

* Extra information will be prefixed by *CADENCE* for debugging purpose.


-----------------
RCS FUNCTIONALITY
-----------------

* For text files, iDMB will try to refresh RCS header and append new log
  messages when a file is fetched or checked out.


------------------
HOW DOES IDMB WORK
------------------

* All commands of IDMB need the user to input the tool name. The IDMB takes the
  tool name and use the tool's search path and policies to execute commands.

* [S] DMS.PATH management: Search path/Working dir/Reparse/Print result

  Based on the parsed result of DMS.PATH file, IDMB returns related information
  of a tool/view.

  Example: Query the search path used by tool "idmb"

	Command: s
	[S]earch path/[W]orking dir/[R]eparse/[P]rint result: s
	Tool Name: (idmb) 

	*IDMS MESSAGE* TOOL: idmb LIB SEARCH PATH: ~cchiu/dms ~cchiu/opus3 
	~cchiu/dms ~cchiu/opus3 

	^^^^^^^^^^^^^^^^^ S ^^^^^^^^^^^^^^^^^

  Example: Query the working directory of tool "idmb" view "sch".

	Command: s
	[S]earch path/[W]orking dir/[R]eparse/[P]rint result: w
	Tool Name: (idmb) 
	View Name: (*) sch

	*IDMS MESSAGE* TOOL: idmb VIEW: sch WORKING DIR: /cadfs7/eda/cchiu/garden/
	/cadfs7/eda/cchiu/garden/

	^^^^^^^^^^^^^^^^^ W ^^^^^^^^^^^^^^^^^

  Example: Reparse the DMS.PATH file and reconfigure the DMS behaviors.

	Command: s
	[S]earch path/[W]orking dir/[R]eparse/[P]rint result: r

* [L] Lock files for edit (co -l)

  Checkout a cellview with a edit lock. If the checked out file is text and has
  valid RCS header and log information resides in, header will be refreshed and
  additional log will be appended.

  Example: Lock all cellviews with 3 characters of cell name.

	Command: l
	Tool Name: (idmb) 
	Lib Name: (*) workLib
	Conifg Name (name/default/none): none
	Cell Name: (*) ???
	View Name: (*) 
	Version Name (pi.si): (current) 

	*IDMS MESSAGE* CHECKOUT ram.icf 2.0 FROM LIB: workLib CONFIG: 
	/cadfs7/eda/cchiu/garden/ram.icf

	*IDMS MESSAGE* CHECKOUT p12.icf 5.0 FROM LIB: workLib CONFIG: 
	/cadfs7/eda/cchiu/garden/p12.icf

	*IDMS MESSAGE* CHECKOUT rom.icf 3.0 FROM LIB: workLib CONFIG: 
	/cadfs7/eda/cchiu/garden/rom.icf

	*IDMS MESSAGE* CHECKOUT sio.icf 2.0 FROM LIB: workLib CONFIG: 
	/cadfs7/eda/cchiu/garden/sio.icf

	*IDMS MESSAGE* CHECKOUT a2d.icf 2.0 FROM LIB: workLib CONFIG: 
	/cadfs7/eda/cchiu/garden/a2d.icf

	*IDMS MESSAGE* TOTAL: 5 CELL(S)
	O.K.

	^^^^^^^^^^^^^^^^^ L ^^^^^^^^^^^^^^^^^

  Note:	The default configuration name will be the one specified in DMS.PATH
	file. If users input "none" as the configuration name, the default one
	will not be used and the most current version will be checkout.

  Note: Any version name specified will over write the configuration mechanism.
